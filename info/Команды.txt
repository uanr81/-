14.1 'infokey' format
=====================

Формат '.infokey' файл легче всего проиллюстрировать на примере.
Например, вот пример файла инициализации, подходящего для поклонников
'vi' или 'less':

     #info
     j       Следующая строка
     k       Предыдущая строка
     l       Следующий-символ
     h       Предыдущий-символ
     \kd     вниз-на-следующую-линию
     \ku     вверх-на-предыдущую-линию
     \       Прокрутка-вперед
     \kD     Прокрутка-вперед-постранично-только
     b       Прокрутка-назад
     \kU     Прокрутка-назад-постранично-только
     g       Начало-узла
     \kh     Начать-от-узла
     G       Конечный-узел
     \ke     Конечный узел
     \t      Выбрать-ссылку-на-линии
     -       История-узла
     n       Следующий-узел
     p       Предыдущий-узел
     u       Верхний-узел
     t       Главный-узел
     d       dir-узел
     \mu     Четко-поиск
     #var
     highlight-searches=On

   Файл состоит из одного или нескольких файлов "sections".  Каждый раздел
   начинается с линией, которая идентифицирует тип раздела. Возможные разделы
   находятся:

'#info'
     Связывание клавиш для информационных окон. Начало этого раздела:
     указывается линией, содержащей '#info' саму на линии.  Если это
     первый раздел в исходном файле, '#info' может быть
     опущен. Остальная часть этого раздела состоит из строк вида:

          STRING пробел ACTION [ пробел [ # коментарий ] ] новая линия

     Пробел - это любая последовательность из одного или нескольких пробелов и/или
     вкладок. Комментарий - любая последовательность любых символов, кроме новой
     строки. STRING Это ключевая последовательность, которая вызывает действие.
     ACTION это имя Info команды.  Символы в STRING интерпретируются
     буквально или с префиксом каретки ('^') для обозначения управляющего символа.
     Символ обратной косой черты, за которым следуют определенные символы, указывает
     следующие комбинации клавиш:

     '\b'
          Backspace
     '\e'
          Escape (ESC)
     '\n'
          Newline
     '\r'
          Return
     '\t'
          Tab
     '\ku'
          Up стрелка
     '\kd'
          Down стрелка
     '\kl'
          Left стрелка
     '\kr'
          Right стрелка
     '\kU'
          Page Up
     '\kD'
          Page Down
     '\kh'
          HOME
     '\ke'
          END
     '\kx'
          Delete (DEL)
     '\mX'
          Meta-X где X любой символ, как описано выше.

     Обратный слэш, сопровождаемый любым другим символом, указывает, что символ
     следует воспринимать буквально. Символы, которым должен предшествовать символ
     обратная косая черта включает каретку, пробел, табуляцию и обратную косую черту
     

'#echo-area'
     Связывание клавиш для эхо-области. Начало этого раздела:
     указывается линией, содержащей '#echo-area' саму по себе. СОДЕРЖАНИЕ
     Остальная часть этого раздела имеет синтаксис, идентичный синтаксису ключа
     Определения для информационной области, описанные выше.

'#var'
     Инициализация переменных.  Начало этого раздела указано
     по строке, содержащей только '#var' саму по себе. Следуя этому
     списку присваиваний переменных, по одному на строку.  Каждая строка состоит
     bp bмени переменной (*note Variables::) с последующим '=' с последующим
     значением. Между именем переменной и пробелом не может быть пробелов.
     Оглавление '=', и все символы, следующие за '=', включая пробелы
     включаются в значение.

   Пустые строки и строки, начинающиеся с '#' игнорируются, за исключением
линии заголовка специального раздела.

   Привязки клавиш, определенные в '.infokey' файле имеют преимущество над GNU
Info привязок клавиш по умолчанию, независимо от того, '--vi-keys' используется. А
привязка ключа по умолчанию может быть отключена путем переопределения ее в
'.infokey' файл с действием 'invalid'. К тому же, _all_ привязки клавиш по
умолчанию можно отключить, добавив эту строку _anywhere_ в соответствующем разделе:

     #stop

   Это заставит GNU Info игнорировать все команды клавиш по умолчанию для
этого раздела.

   Остерегайтесь: '#stop' Может быть опасным. Поскольку он отключает все клавиши
   привязок по умолчанию, вы должны предоставить достаточно новых привязок клавиш,
   чтобы включить все Необходимые действия. Неспособность привязать любой ключ к
   'quit' команде, например, может привести к расстройству.

   Порядок, в котором привязки клавиш определены в '.infokey' Файл не важно, за
   исключением того, что сводка команд, 'get-help-window' команда отображает только
   _first_ key который привязан к каждой команде.
